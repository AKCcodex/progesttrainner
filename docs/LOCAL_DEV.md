# Local development guide

## 1. Prerequisites

- Docker + Docker Compose (preferred)
- For non-Docker backend dev: Python 3.12, PostgreSQL 16, Redis 7
- An AI provider — Ollama Cloud, local Ollama, or any of OpenAI / Anthropic / Gemini / OpenAI-compatible

## 2. Configure environment

```bash
cp .env.example .env
```

Minimum required values:

```env
AI_PROVIDER=ollama
OLLAMA_BASE_URL=http://host.docker.internal:11434   # local Ollama
OLLAMA_MODEL=gemma3:4b
JWT_SECRET=<random 64 chars>
INTERNAL_BOT_TOKEN=<random 64 chars>
```

If you're running Ollama **locally on the host**, `host.docker.internal:11434` works on macOS/Windows; on Linux add `extra_hosts: - "host.docker.internal:host-gateway"` to the backend service in `docker-compose.yml`.

For **Ollama Cloud**:

```env
OLLAMA_BASE_URL=https://ollama.com
OLLAMA_API_KEY=<your key>
OLLAMA_MODEL=gemma3:4b
```

For **OpenAI**:

```env
AI_PROVIDER=openai
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4o-mini
```

## 3. Boot the stack

```bash
docker compose up -d --build
docker compose logs -f backend worker
```

Wait until you see:

```
INFO | uvicorn.error | Application startup complete.
INFO | app.scheduler.runner | scheduler started with 6 jobs
```

Then verify:

```bash
curl http://localhost:8000/health
# → {"status":"ok"}
```

The frontend is served at `http://localhost:3000`.

## 4. First run

1. Open `http://localhost:3000`, click **Register**, create your account.
2. Go to **Goals → New goal**, set a title and a daily minute budget.
3. Open the goal, click **+ Add resource**, paste a YouTube URL or article link (or upload a PDF / type a note).
4. The `backend` logs will show resource enrichment:
   ```
   INFO | app.integrations.youtube.transcripts | falling back to yt-dlp for <id>
   ```
5. The scheduler generates today's lessons at **06:00 UTC** by default. To generate them immediately:
   ```bash
   docker compose exec backend python -c "
   import asyncio
   from app.database.session import SessionLocal
   from app.services.lesson_service import LessonService
   from datetime import date
   async def go():
       db = SessionLocal()
       try:
           n = await LessonService(db).generate_today_for_all_users(date.today())
           print('generated', n, 'lessons')
       finally:
           db.close()
   asyncio.run(go())"
   ```
6. Reload the goal page — lessons appear with a **Take quiz** button. The quiz is generated by the AI from your resources' transcripts / content.

## 5. Telegram bot

1. Create a bot with [@BotFather](https://t.me/BotFather). Copy the token to `TELEGRAM_BOT_TOKEN` in `.env`.
2. In the web app: open your profile and click **Generate Telegram link code** (TODO: this is the `/api/v1/users/me/telegram-link-code` endpoint; wire it into the profile page if it's not yet).
3. In Telegram, send `/start` then `/link <code>`.
4. Use `/today` and tap the inline buttons to mark lessons done.

## 6. Running the backend without Docker

```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt -r requirements-dev.txt
export DATABASE_URL=postgresql+psycopg://coach:coach@localhost:5432/coach
export REDIS_URL=redis://localhost:6379/0
export OLLAMA_BASE_URL=http://localhost:11434
export OLLAMA_MODEL=gemma3:4b
alembic upgrade head
uvicorn app.main:app --reload --port 8000
# in a second terminal:
python -m app.workers.rq_worker
```

## 7. Common issues

| Symptom | Cause | Fix |
|---------|-------|-----|
| `alembic upgrade head` hangs in Docker | Postgres not yet ready | the entrypoint script retries; check `docker compose logs backend` |
| `/health` returns 200 but all API calls fail with 401 | stale token | clear localStorage and re-login |
| Telegram bot says "not linked" after `/start` | link code expired (5 min) | generate a new code, re-send |
| Lessons never appear | scheduler hasn't run (06:00 UTC) | use the manual-generation snippet above |
| AI returns 502 from `/internal/ai/test` | model name wrong, key missing, or base URL wrong | verify env vars, hit the provider's docs directly |

## 8. Production notes

- **Single instance** is the current design — APScheduler is in-memory and the RQ worker is single-process. To scale horizontally, move scheduled jobs into RQ-cron or a dedicated beat process.
- **Secrets** — rotate `JWT_SECRET` and `INTERNAL_BOT_TOKEN` before going live; both are read from env.
- **Backups** — `pg_dump` regularly. PDFs in `data/pdfs/` should be backed up too if you want to re-extract transcripts.
- **HTTPS** — terminate TLS at a reverse proxy (Caddy / nginx / Traefik). The backend container listens on plain HTTP.
